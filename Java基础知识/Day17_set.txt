一、登录注册案例

按照如下操作
	A：有哪些类
	B：每个类有哪些东西？
	C：类与类之间的关系

 分析：
	A：有哪些类？
		用户类
		测试类
	B：每个类有哪些东西？
	   	用户类
			成员变量：用户名，密码
			构造方法; 无参构造
			成员方法：getXXX() 和setXXX( )
				 登录，注册、、
		
		假如用户类的内容比较多，将来进行维护比较麻烦
			用户基本信息的操作类
				成员变量：用户名，密码
				构造方法; 无参构造
				成员方法：getXXX() 和setXXX( )
			用户操作类
				登录，注册
		
		测试类：
			main方法
	C；类之间有什么关系?
		在测试类中创建用户操作类和用户基本描述类的对象，并且使用功能



分包：
	A：功能划分
	B：模块划分
	C: 先按模块划分，再按功能划分

按功能进行划分
	用户基本描述类 ; 其实就是用户基本信息的描述
	用户操作接口:  
	用户操作实现:
		先是集合实现，Io流实现，GUI用户界面实现，数据库实现
